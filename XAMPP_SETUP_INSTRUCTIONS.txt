MEGA SCHOOL PLAZA - XAMPP SETUP INSTRUCTIONS
============================================

REQUIREMENTS:
- XAMPP with PHP 7.4+ and MySQL
- Web browser
- Text editor (optional)

INSTALLATION STEPS:

1. DOWNLOAD AND EXTRACT
   - Download the ZIP file from v0
   - Extract to your XAMPP htdocs folder
   - Rename the folder to 'megaplaza' (or your preferred name)
   - Final path should be: C:\xampp\htdocs\megaplaza\ (Windows) or /opt/lampp/htdocs/megaplaza/ (Linux)

2. START XAMPP SERVICES
   - Open XAMPP Control Panel
   - Start Apache and MySQL services
   - Ensure both services are running (green status)

3. CREATE DATABASE
   - Open web browser and go to: http://localhost/phpmyadmin
   - Click "New" to create a new database
   - Name it 'megaplaza'
   - Click "Create"
   - Select the 'megaplaza' database
   - Click "Import" tab
   - Choose file: database/megaplaza_complete.sql
   - Click "Go" to import

4. CONFIGURE SETTINGS
   - Open config/database.php
   - Verify database settings:
     * DB_HOST: localhost
     * DB_NAME: megaplaza
     * DB_USER: root
     * DB_PASS: (leave empty for default XAMPP)
   
   - Open config/config.php
   - Update SITE_URL to match your setup:
     * For default: http://localhost/megaplaza
     * If using different folder name: http://localhost/your-folder-name

5. SET PERMISSIONS (Linux/Mac only)
   - Open terminal in the project directory
   - Run: chmod -R 755 uploads/
   - Run: chmod -R 755 config/

6. ACCESS THE SYSTEM
   - Open web browser
   - Go to: http://localhost/megaplaza
   - You should see the landing page

TROUBLESHOOTING:
================

1. "Database connection failed" error:
   - Check if MySQL service is running in XAMPP
   - Verify database name and credentials in config/database.php
   - Ensure database was imported correctly

2. "Page not found" errors:
   - Check if Apache service is running
   - Verify the correct URL path
   - Ensure files are in the correct htdocs directory

3. File upload issues:
   - Check if uploads/ directory exists and has write permissions
   - Verify MAX_FILE_SIZE in config/config.php
   - Check PHP upload settings in php.ini

4. Session issues:
   - Clear browser cookies and cache
   - Check if session.save_path is writable in PHP configuration

5. Permission denied errors (Linux/Mac):
   - Run: sudo chmod -R 755 /opt/lampp/htdocs/megaplaza/
   - Run: sudo chown -R www-data:www-data /opt/lampp/htdocs/megaplaza/
